10-Layer Microservices Architecture

From electron flow to token minting — each layer handles a distinct operational concern

L1 Physics Engine

The "Green Audit" — verifies kWh dispensed vs. kWh received via telematics. Variance threshold <15%.

/services/01-physics-engine /packages/migrid-physics
L2 Grid Signal

OpenADR 3.0 Virtual End Node (VEN) for utility demand response communication and price signals.

/services/02-grid-signal /packages/openadr-client
L3 VPP Aggregator

Aggregates EVs and BESS to bid capacity into wholesale markets as a Virtual Power Plant.

/services/03-vpp-aggregator
L4 Market Gateway

Arbitrage engine connecting to CAISO, PJM, and Nord Pool wholesale energy markets.

/services/04-market-gateway
L5 Driver DX

Smart routing, voice commands, and real-time charger availability for drivers.

/services/05-driver-bff /apps/driver-app-mobile
L6 Engagement

CRM, targeted banners, contextual support ticketing, and driver communication.

/services/06-engagement
L7 Device Gateway

Hardware abstraction layer for OCPP 1.6/2.0.1, OCPI roaming, and ISO 15118 Plug&Charge.

/services/07-device-gateway /packages/iso15118-crypto
L8 Energy Manager

Dynamic Load Management (DLM) and site safety. Edge-ready for offline operation.

/services/08-energy-manager /apps/admin-portal-web
L9 Commerce Engine

Flexible billing, tariffs, subscriptions, and split-billing for fleet operators.

/services/09-commerce-engine
L10 Token Bridge

Rewards and staking via the Open-Wallet Framework. Hybrid SQL + blockchain ledger.

/services/10-token-engine /scripts/migrations

Quick Start

Get MiGrid running locally in under 5 minutes

# Clone the repository
git clone https://github.com/migrid-org/migrid-core.git
cd migrid-core

# Start infrastructure (TimescaleDB, Redis, Gateway services)
docker-compose up -d

# Seed vehicle physics models (F-150 Lightning, Rivian R1T, etc.)
npm run seed:physics

# Verify services are running
curl http://localhost:3000/health
# {"status": "ok", "services": ["physics", "device", "commerce"]}
